================================================================================
SECTION 5: KEY CONCEPTS TO REMEMBER
================================================================================

1. Token Bucket = bucket with max capacity, tokens added at fixed rate,
   each request consumes tokens, no tokens = rejected.

2. Lazy Refill = no background thread, calculate tokens on each call
   using elapsed time. Simpler, no scheduling overhead.

3. Thread-safety = ReentrantLock protects shared mutable state
   (currentTokens, lastRefillTimestamp).

4. Why not AtomicInteger for currentTokens?
   Because read + check + print + decrement must happen atomically
   as a group. AtomicInteger only protects individual operations.

5. System.nanoTime() for elapsed time (not currentTimeMillis)
   because it's not affected by clock changes (NTP, daylight saving).

6. double for currentTokens because elapsed time can be fractional
   (0.3 sec x 5 rate = 1.5 tokens).
