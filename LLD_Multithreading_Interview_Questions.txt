SECTION 9: TIPS FOR 6+ YOE INTERVIEWS
================================================================================

1. Always start with REQUIREMENTS CLARIFICATION — ask questions before coding.
2. Identify ENTITIES and their relationships first (draw a class diagram mentally).
3. Apply SOLID principles — interviewers at senior level look for clean abstractions.
4. Use DESIGN PATTERNS where natural — don't force them. Explain WHY you chose one.
5. For multithreading: always discuss THREAD SAFETY, what shared state exists,
   which data structure is appropriate, and potential race conditions.
6. Discuss TRADE-OFFS — there is no single correct answer.
7. Think about EXTENSIBILITY — how would you add a new feature?
8. Write CLEAN, READABLE code — use meaningful names, keep methods small.
9. Handle EDGE CASES and ERROR scenarios proactively.
10. At 6+ YOE, you're expected to LEAD the design, not be guided.
